# YouTube Downloader & Transcriber CLI - AI Agent Usage Guide

## Overview
This CLI tool downloads YouTube videos, converts them to audio, transcribes with Whisper, and organizes all output files. Designed for use by AI agents in automated workflows.

## Prerequisites

### System Tools (Install via Homebrew)
```bash
brew install ffmpeg
brew install yt-dlp
```

### Runtime Requirements
- **Bun** runtime must be used (not Node.js or Deno)
- macOS arm64 architecture supported

## Installation

1. Install NPM dependencies:
```bash
bun add commander @xenova/transformers open
```

2. Verify installation:
```bash
bun run ./cli.ts --help
```

## CLI Usage Examples

### Basic Download & Transcription
```bash
# Single video download and transcription
bun run ./cli.ts --download https://www.youtube.com/watch?v=D3yMC_qoAes
```

### Multiple Videos (Space-separated URLs)
```bash
# Process multiple videos in one command
bun run ./cli.ts --download "https://youtube.com/watch?v=1" "https://youtube.com/watch?v=2"
```

### Custom Output Directory
```bash
# Specify custom output folder
bun run ./cli.ts --download https://www.youtube.com/watch?v=D3yMC_qoAes --output ./my-info
```

### Help & Documentation
```bash
# Show help message
bun run ./cli.ts --help
```

## Output Structure

Each video creates a subfolder named after the YouTube Video ID:

```
./output/
└── [video-id]/
    ├── metadata-info.md      # Video metadata in Markdown format
    ├── video.mp4             # Downloaded MP4 video file
    ├── audio.wav             # Converted WAV audio (16kHz, mono)
    ├── transcript.txt        # Line-by-line transcript with timestamps
    ├── raw.json              # Raw Whisper transcription data
    └── captions.srt          # SRT caption file for subtitles
```

## File Descriptions

| File | Description |
|------|-------------|
| `metadata-info.md` | Video metadata including title, duration, uploader |
| `video.mp4` | Original downloaded video in MP4 format |
| `audio.wav` | Audio extracted from video (16kHz PCM) |
| `transcript.txt` | Human-readable transcript with timestamps |
| `raw.json` | Raw Whisper output with timing data |
| `captions.srt` | SRT subtitle file for video players |

## Workflow Steps

1. **Download** - Uses yt-dlp to download MP4 and metadata
2. **Convert** - ffmpeg converts MP4 to WAV audio (16kHz, mono)
3. **Transcribe** - Whisper via transformers.js generates text with timing
4. **Generate** - Creates SRT captions from transcription data
5. **Organize** - Files organized in video ID subfolders
6. **Open** - Output folder opens automatically when complete

## Error Handling

- Invalid YouTube URLs will be rejected with error message
- Missing system tools (ffmpeg, yt-dlp) will cause installation errors
- Network failures during download will log errors and continue to next video
- Whisper model downloads are cached locally after first use

## AI Agent Integration Tips

1. **Batch Processing**: Pass multiple URLs separated by spaces for efficient batch processing
2. **Output Organization**: Use unique output directories per project or campaign
3. **Logging**: Check `[INFO]`, `[WARN]`, `[ERROR]` prefixes in console output
4. **Verification**: Always verify `metadata-info.md` contains correct video information
5. **Cleanup**: Output folders can be safely deleted after processing

## Troubleshooting

### "ffmpeg not found" error
```bash
brew install ffmpeg
```

### "yt-dlp not found" error
```bash
brew install yt-dlp
```

### Whisper model download slow
- First run downloads ~50MB model, subsequent runs use cache
- Ensure stable internet connection for initial model download

### Output folder doesn't open automatically
- Check macOS security settings allow `open` command execution
- Manually navigate to `./output/[video-id]/` directory
