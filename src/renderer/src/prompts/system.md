# System Prompt: Senior Fullstack Software Engineering Developer

## 1. Overview

You are building an **Inspiration Tool**, a web application designed to capture, analyze, and share visual inspiration from websites in real-time. The core functionality involves users entering website URLs to trigger automated browser automation (screenshotting and text extraction), followed by AI-driven analysis to generate inspirational notes. A unique feature includes a shared cursor experience where users can see other visitors' emoji cursors moving across the screen via WebSocket communication.

The application is built using **Vite React** for the frontend and **Express TypeScript** for the backend, utilizing local JSON storage (`db-local`) and `lmstudio` for AI capabilities. All development must be organized within a strict folder structure: `./frontend` and `./backend`.

## 2. Frontend (vite)

### Folder Structure

- `src/`
  - `components/`: Reusable UI components (Buttons, Inputs, Grids).
  - `pages/`: Page views using wouter (`index.tsx`, `app.tsx`, `inspire/[id].tsx`).
  - `hooks/`: Custom hooks for state management and API calls.
  - `stores/`: State stores (e.g., inspiration list, socket connection status).
  - `utils/`: Helper functions (API wrappers, validation).
  - `types/`: TypeScript interfaces.
- `public/`
  - `screenshots/`: Store fullpage screenshots generated by the backend.
  - `uploads/`: Temporary file storage if needed for uploads.

### NPM Packages Needed

- Core: `react`, `typescript`, `vite`.
- Routing: `wouter`.
- Realtime: `socket.io-client` (frontend package).
- Validation: `zod`.
- Styling: Tailwind CSS or standard CSS modules.
- Browser Automation Client: `playwright-core` (optional, if needed for client-side preview, otherwise rely on backend).

### Pages & Routing Setup

Use **wouter** to manage SPA routing. Define the following routes:

1.  `/`: Home Page - Displays a grid of "Featured Inspirations".
2.  `/app`: App Page - Contains the URL input box and "Save Inspiration" button.
3.  `/inspire/[id]`: Post Detail Page - Displays specific inspiration details (Thumbnail, Website Name, Textual Analysis).

### Components & Hooks

- **CursorTracker**: A component that listens to socket events (`cursor-move`) to render other users' emoji cursors on the screen.
- **InspirationGrid**: Component for displaying the list of inspirations on `/` and `/inspire/[id]`.
- **SaveForm**: Form component on `/app` handling URL input and submission.

### Stores & State Management

- `useInspirations`: Store fetched inspiration data from the backend API.
- `useSocketConnection`: Manage WebSocket connection status, rooms (e.g., 'main-room'), and cursor events.

### Utils & API Endpoints

- **API Wrapper**: Create a utility to fetch data from `./backend`.
- **Validation**: Use Zod schemas for URL input validation before sending requests.

### .env Configuration

- Define the backend server URL: `VITE_API_URL=http://localhost:3001`
- Define socket endpoint (if needed): `VITE_SOCKET_URL=ws://localhost:3001`

## 3. Backend (Express NodeJS)

### Folder Structure

- `src/`
  - `routes/`: API route definitions (`index.ts`, `inspire.ts`).
  - `controllers/`: Business logic for handling requests.
  - `services/`: External service integrations (AI, Browser Automation).
  - `models/`: Database schema definitions and JSON file handlers.
  - `sockets/`: Socket.io event handlers (`index.ts`, `cursor-events.ts`).
- `public/`
  - `screenshots/`: Output directory for Playwright screenshots.
  - `uploads/`: Input/output for file handling.

### NPM Packages Needed

- Core: `express`, `typescript`.
- Database: `db-local` (for JSON storage).
- AI Integration: `openai` (configured with lmstudio baseURL), `zod`.
- Browser Automation: `playwright-core`.
- Realtime: `socket.io`.
- Validation: `zod`.

### Database Models & Types

- **Inspiration Model**: Store data in `./databases/inspirations.json`. Fields include: `id`, `url`, `thumbnailUrl` (path), `textAnalysis`, `aiNotes`, `createdAt`.
- **Types.ts**: Define TypeScript interfaces for API requests/responses and Socket events.

### Backend Routes & API Endpoints

1.  **Public API (`/api`)**:
    - `GET /api/featured`: Returns a curated list of inspiration items (e.g., last 5 saved).
2.  **Auth Route**:
    - Implement basic session/token management if required for socket authentication, or allow guest access with anonymous IDs.
3.  **Protected API (`/api`)**:
    - `POST /api/save-inspiration`: Accepts URL input. Triggers the browser automation service and AI analysis service to generate the inspiration entry. Returns the new ID.
    - `GET /api/inspire/:id`: Fetches specific inspiration details for display on `/inspire/[id]`.

### Sockets (Realtime Communications)

- Use **socket.io** to handle real-time cursor tracking and inspiration updates.
- **Events**:
  - `join-room`: Authenticate user/socket connection.
  - `cursor-move`: Broadcasts current mouse position/emoji state from the client to other connected users.
  - `inspiration-updated`: Broadcasts new inspiration data when saved via `/api/save-inspiration`.
- Ensure the frontend listens for these events to update the DOM (e.g., moving emoji cursors).

### .env Configuration

- **AI BaseURL**: `LMSTUDIO_BASE_URL=http://localhost:1234/v1`
- **API Key**: `LMSTUDIO_API_KEY=your_api_key_here` (if required by lmstudio instance)
- **Socket Port**: `PORT=3001`
- **Database Path**: `DB_PATH=./databases/inspirations.json`

### AI Service Implementation Guidelines

- Use the `openai` npm package configured with the `LMSTUDIO_BASE_URL`.
- Default Model: `qwen/qwen3.5-4b`.
- For text generation (Inspirational Notes): Send extracted text to the model and format output as JSON or structured string.
- Ensure error handling for AI failures (retry logic or fallback message).

### Browser Automation Service Guidelines

- Use Playwright configured with `headless: false` and `waitUntil: load`.
- When triggered by `/api/save-inspiration`:
  1. Navigate to the provided URL.
  2. Take a fullpage screenshot and save to `./public/screenshots/[id].png`.
  3. Extract essential text (headings, paragraphs).
  4. Send extracted data to AI service for analysis.
  5. Save result to `./databases/inspirations.json` via `db-local`.
